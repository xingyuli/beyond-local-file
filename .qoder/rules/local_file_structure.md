# local-file/agentic Directory Convention

This file defines the storage specification for Qoder-generated content.

## Directory Structure

```
local-file/agentic/
├── analysis/    # Technical analysis reports generated by Qoder (on-demand, one file per report)
├── drafts/      # Drafts written by Qoder: design proposals, document drafts, OpenAPI specs, etc.
└── specs/       # Formal specification files (technical specs, API specs, requirements docs)
```

## Directory Usage

| Path | When to Write | Written By |
|------|---------------|------------|
| `agentic/analysis/` | After technical analysis tasks complete | Qoder (on-demand) |
| `agentic/drafts/` | After design proposals, document drafts, or spec drafts complete | Qoder |
| `agentic/specs/` | After formal specification documents complete | Qoder |

## Notes

- `agentic/analysis/` stores technical analysis reports, with each report as a separate file. Filenames should clearly describe the analysis topic.
- `agentic/drafts/` stores various draft documents, including but not limited to design proposals, document drafts, and API specifications.
- `agentic/specs/` stores formal specification files such as technical specifications, API specifications, and requirements documents.
- All Qoder-generated analysis, draft, and specification documents must be placed in their corresponding subdirectories. Do not scatter files in the `local-file/agentic/` root directory.
