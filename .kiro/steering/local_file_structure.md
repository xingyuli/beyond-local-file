# local-file/kiro Directory Convention

This file defines the storage specification for Kiro-generated content.

## Directory Structure

```
local-file/kiro/
├── analysis/    # Technical analysis reports generated by Kiro (on-demand, one file per report)
└── drafts/      # Drafts written by Kiro: design proposals, document drafts, OpenAPI specs, etc.
```

## Directory Usage

| Path | When to Write | Written By |
|------|---------------|------------|
| `kiro/analysis/` | After technical analysis tasks complete | Kiro (on-demand) |
| `kiro/drafts/` | After design proposals, document drafts, or spec drafts complete | Kiro |

## Notes

- `kiro/analysis/` stores technical analysis reports, with each report as a separate file. Filenames should clearly describe the analysis topic.
- `kiro/drafts/` stores various draft documents, including but not limited to design proposals, document drafts, and API specifications.
- All Kiro-generated analysis and draft documents must be placed in their corresponding subdirectories. Do not scatter files in the `local-file/kiro/` root directory.
