# local-file/agentic Directory Convention

This file defines the storage specification for Kiro-generated content.

## Directory Structure

```
local-file/agentic/
├── analysis/    # Technical analysis reports generated by Kiro (on-demand, one file per report)
├── drafts/      # Drafts written by Kiro: design proposals, document drafts, OpenAPI specs, etc.
└── specs/       # Formal specification files (technical specs, API specs, requirements docs)
```

## Directory Usage

| Path | When to Write | Written By |
|------|---------------|------------|
| `agentic/analysis/` | After technical analysis tasks complete | Kiro (on-demand) |
| `agentic/drafts/` | After design proposals, document drafts, or spec drafts complete | Kiro |
| `agentic/specs/` | After formal specification documents complete | Kiro |

## Notes

- `agentic/analysis/` stores technical analysis reports, with each report as a separate file. Filenames should clearly describe the analysis topic.
- `agentic/drafts/` stores various draft documents, including but not limited to design proposals, document drafts, and API specifications.
- `agentic/specs/` stores formal specification files such as technical specifications, API specifications, and requirements documents.
- All Kiro-generated analysis, draft, and specification documents must be placed in their corresponding subdirectories. Do not scatter files in the `local-file/agentic/` root directory.
